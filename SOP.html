1 System Overview

What is BulkImport and how does it fit into your workflow

The BulkImport System is an enterprise-grade web application that enables companies to import, manage, and synchronize bulk master data (Ledger, Item, HSN, Spare Parts, Tools) into MS SQL Server databases from Excel files. It provides a multi-tenant architecture supporting multiple companies from a single deployment.

📥 Bulk Excel Import

Upload structured Excel files and import thousands of records with built-in validation and duplicate checking.

🏢 Multi-Tenant

Each company gets an isolated database connection managed through secure JWT session tokens.

🔐 Two-Step Auth

Company-level login followed by user-level login ensures layered security for all data operations.

📊 Master Data Mgmt

Full CRUD operations on Ledger, Item, HSN, Spare Part, and Tool masters with AG Grid inline editing.

📦 Stock Uploads

Bulk upload and update stock quantities for items, spare parts, and tools via Excel.

🛡️ Role-Based Access

Customer vs Indus (Admin) login types each unlock different features and pages.

5 Stock Upload

Bulk update inventory/stock quantities via Excel for Items, Spare Parts, and Tools

The Stock Upload module allows you to update existing stock levels in bulk. It does not create new records — it updates quantity fields on existing master records.

📦 Item Stock Upload

Update stock quantities, opening balances, and warehouse/bin locations for inventory items.

🔧 Spare Part Stock Upload

Bulk update spare part quantities across storage locations.

🛠️ Tool Stock Upload

Update tool availability counts and stock statuses.

Stock Upload Workflow

Select Module Type

Choose Item Stock Upload, Spare Part Stock Upload, or Tool Stock Upload from the dropdown.

Select Group (if applicable)

For Item and Tool stock, select the relevant Group to filter records and load the data table.

Upload Excel or Edit Inline

Upload a pre-filled Excel file with the new stock quantities, or edit quantities inline in the data grid.

Review & Confirm

Review all changes. The system will prompt a confirmation dialog before applying updates.

Submit

Stock levels are updated in the database. A success message confirms the number of records updated.

⚠️

Data Loss Warning
If you switch to a different module type while data is loaded, the system will warn you that unsaved changes will be lost. Always save or discard before switching.

9 ERP Transaction Delete

Securely delete master data and transaction records with multi-factor authorization

🚨

Irreversible Operation Warning
Transaction Delete permanently removes data from the system. This action cannot be undone. Ensure you have proper authorization and a valid business reason before proceeding.

The ERP Transaction Delete feature offers two modes:

📋 Master Data Delete

Soft-delete individual master records (Ledger, Item, etc.) that are no longer in use. The record is flagged as deleted — not physically removed.

💣 Transaction Delete

Hard-delete entire transaction datasets for a selected module with CAPTCHA-verified multi-step confirmation.

Master Data Delete Workflow

Select Module & Sub-Module

Choose the master type (e.g., Ledger Master) and its sub-module (e.g., a specific Ledger Group).

Check Usage

The system checks if the selected master is referenced in any transactions. If in use, deletion is blocked.

Enter Credentials & Reason

Enter your Username, Password, and a mandatory Reason for the deletion. This is logged for audit purposes.

Confirm Soft Delete

Click Delete. The record is flagged with isDeletedTransaction = true but remains in the database.

Transaction Delete Workflow

Select Module & Sub-Module

Choose the target module and sub-module whose transactions you want to permanently delete.

Enter Credentials & Reason

Enter Username, Password, and a detailed Reason. These are logged in the activity audit trail.

Solve CAPTCHA

Solve a simple arithmetic CAPTCHA (e.g., "7 + 4 = ?") to prevent accidental submissions.

Monitor Progress

A progress bar tracks deletion across each related table. Per-table progress percentages are displayed in real time.

View Result

A success popup shows the total number of records deleted across all tables. Close to return to the form.

11 Module Group Authority

Admin feature — batch-assign module groups to companies

ℹ️

Indus Login Required
This feature is only available to Indus admin users.

Module Group Authority allows administrators to create logical groups of modules and apply those groups to client companies in one operation — instead of assigning modules one by one.

Create a Module Group

Define a named group (e.g., "Standard ERP Package") to which you will assign modules.

Add Modules to the Group

Select modules from the available module list and add them to the group. Remove modules as needed.

Apply Group to Client

Select the target company and apply the group. All modules in the group are instantly assigned to that company.

12 Troubleshooting

Common issues and their resolutions

IssueLikely CauseResolution
"No connection string" error JWT token missing sessionId claim or session expired Log out and log back in. Complete both steps of authentication.
401 Unauthorized — auto redirect to login JWT token expired (default 8 hours) Re-login. The system automatically redirects on token expiry.
Excel import fails with "invalid file" Wrong file extension or filename mismatch Ensure file is .xlsx format and filename exactly matches the module name.
Import shows all rows as Duplicate Records already imported previously Use the Clear All Data option (admin) or delete existing records before re-importing.
CORS error in browser console Frontend origin not in allowed list Verify VITE_API_BASE_URL and ensure the origin is in Program.cs CORS policy.
Stock upload: "Group not found" Selected group has no records or was deleted Check master data for the group. Re-import group records if missing.
Module not appearing in sidebar Module not assigned to company or inactive Check Module Authority. Ensure the module is active and has correct Group Index.
Company Subscription page blank (Indus) IndusConnection string in appsettings.json invalid Verify the IndusConnection connection string points to the correct admin database.
Backup transfer stalls at 0% SQL Server backup path not accessible from API server Use UNC paths or ensure API and SQL Server share filesystem access.

13 Frequently Asked Questions

Quick answers to common questions

Q: What Excel format does the system require?

A: Only .xlsx files are supported. The filename must exactly match the module name (e.g., LedgerMaster.xlsx). The first row must be the column headers, and they must match the expected schema for that master type.

Q: Can I import data for multiple companies?

A: Yes, but you must log in separately for each company. The system uses per-session tenant isolation — data imported in one company session never affects another.

Q: What happens when I delete a record using ERP Transaction Delete?

A: Master Data Delete performs a soft delete — the record is flagged as deleted but remains in the database for audit. Transaction Delete is a hard delete — rows are permanently removed from all related tables.

Q: How many records can I import at once?

A: The system supports files up to 500 MB. Practical limits depend on the number of columns and validation complexity, but tens of thousands of rows per file are supported.

Q: What is the difference between Customer Login and Indus Login?

A: Customer Login is for company-specific users who manage their own data (imports, masters, stock). Indus Login is for system administrators who manage company subscriptions, module groups, and cross-company operations.

Q: How do I add a new module to the system?

A: Go to Module Authority → Click Create Module → Fill in Module Name, Display Name, Head Name, Group Index, and Display Order → Save. The module appears in the sidebar and becomes available for import operations.

Q: Is the system accessible on mobile devices?

A: Yes. The frontend is responsive — the sidebar collapses to an overlay on small screens. However, for bulk data operations involving large grids, a desktop or laptop is recommended for optimal experience.

Q: Where can I see a log of all past imports and changes?

A: Indus users can access the Activity Log from the user profile dropdown in the header. It shows all Insert, Update, Delete, and Clear operations across the system with timestamps and user details.
